# 家長線上回條 — 後端部署說明（約 5 分鐘）

回條資料會寫進你自己的 Google 試算表，老師打開試算表就能看。**請用學校帳號 `ipad@mail2.smes.tyc.edu.tw` 操作**。

## 步驟

1. **建立試算表**
   用 `ipad@` 開一份新的 Google 試算表，命名為「第103屆畢業典禮-家長回條」。

2. **開啟 Apps Script**
   在試算表上方選單：**擴充功能 → Apps Script**。

3. **貼上後端程式**
   - 把編輯器裡預設的 `Code.gs` 內容**全部刪掉**，貼上本資料夾 [`Code.gs`](Code.gs) 的內容。
   - （建議）左側齒輪 ⚙ **專案設定 → 勾選「在編輯器顯示 appsscript.json」**，回到編輯器把 `appsscript.json` 內容換成本資料夾的版本。
   - 存檔（💾）。

4. **部署為網頁應用程式**
   右上 **部署 → 新增部署 → 類型選「網頁應用程式」**：
   - 說明：`回條 v1`
   - 執行身分：**我（ipad@…）**
   - 具有存取權的使用者：**任何人**
   - 按 **部署**。

5. **授權（第一次必做）**
   部署時會跳授權 → 選 `ipad@` 帳號 → 「進階」→「前往專案（不安全）」→「允許」。
   （或回編輯器，函數下拉選 `authorize` → 按「執行」走完同樣授權。）

6. **複製網址給阿凱**
   部署完成後會顯示「網頁應用程式」網址，長得像：
   `https://script.google.com/macros/s/AKfyc..../exec`
   **把這串 /exec 網址貼給阿凱**（或自行填入 `script.js` 裡的 `RSVP_ENDPOINT`），表單就會上線。

## 完成後

- 家長在網站「線上回條」送出 → 試算表「回條」分頁自動新增一列（送出時間／班級／姓名／人數／悄悄話）。
- 要看回覆，直接打開那份 Google 試算表即可。
- 日後若想改部署，**請用「管理部署 → 編輯（鉛筆）→ 版本：新版本」**更新，網址不變。

> 小提醒：測試時用無痕視窗開網站，避免多個 Google 帳號登入造成的權限混亂。

## 🆕 更新後端：祝福牆審核（從 v1.8.0 起）

新版 `Code.gs` 多了「祝福牆」功能——你審核通過的祝福會出現在網站前端跑馬燈與卡片牆。

### 更新步驟

1. 打開試算表 → **擴充功能 → Apps Script**。
2. 把 [`Code.gs`](Code.gs) 內容**整個換成新版**、存檔。
3. **重新部署成新版本**（網址不變）：**部署 → 管理部署 → 點現有部署的鉛筆 ✏ → 版本選「新版本」→ 部署**。
4. （若跳授權）選 `ipad@` → 進階 → 前往專案 → 允許。

### 怎麼審核（讓祝福公開到前端）

- 打開「回條」分頁，會多一欄 **「公開」(F 欄)**。
- 想公開哪一則祝福，就在那一列的 **F 欄填 `1`**（或打勾、填「是」）。
- 想撤下：把 F 欄清空即可。
- 前端每隔幾分鐘自動更新；家長也可重新整理看到最新祝福牆。

> 前端顯示時，學生姓名會自動**遮罩**（例：六年6班 王○○）保護隱私；完整姓名只有你在試算表看得到。

## 🔔 更新後端：新祝福 LINE 即時通知（從 v1.10 起）

有家長送出新祝福時，後端會自動推一張 **LINE 卡片**給你（含班級／畢業生／祝福內容／累計則數）；萬一寫入失敗也會推一張紅色告警卡，讓你即時掌握。憑證**不寫在程式碼裡**，改放「指令碼屬性」，所以 `Code.gs` 可以安全公開。

### 設定步驟（一次性，約 3 分鐘）

1. 打開試算表 → **擴充功能 → Apps Script**，把 [`Code.gs`](Code.gs) 內容**整個換成新版**、存檔。

2. **設定 LINE 憑證**（兩種方式擇一）：

   **方式 A — 指令碼屬性 UI（推薦，最安全）**
   - 左側齒輪 ⚙ **專案設定** → 捲到最下方 **指令碼屬性** → **新增指令碼屬性**，新增兩筆：

     | 屬性 (Property) | 值 (Value) |
     |---|---|
     | `LINE_CHANNEL_ACCESS_TOKEN` | （阿凱提供的 Channel Access Token） |
     | `LINE_ADMIN_USER_ID` | （阿凱提供的 LINE userId，U 開頭） |
   - 按 **儲存指令碼屬性**。

   **方式 B — 用 setupLineNotify() 函數**
   - 在 `Code.gs` 最下方 `setupLineNotify()` 裡，把 `TOKEN` / `USER_ID` 兩個空字串填上你的憑證。
   - 上方函數下拉選 `setupLineNotify` → 按 **執行**。
   - 執行成功後**務必把那兩個值改回空字串**並存檔（避免憑證殘留在程式碼）。

3. **重新部署成新版本**（網址不變）：**部署 → 管理部署 → 鉛筆 ✏ → 版本選「新版本」→ 部署**。

4. **重新授權（重要）**：這版多了「對外連線」權限（呼叫 LINE API），第一次會跳新的授權畫面 → 選 `ipad@` → 進階 → 前往專案 → **允許**。
   - 若沒自動跳，回編輯器函數下拉選 `authorize` → 按「執行」走完授權。

5. **測試**：函數下拉選 `testLineNotify` → 按「執行」，你的 LINE 應立刻收到一張「收到一則新的畢業祝福」測試卡片，代表大功告成。

### 小提醒

- LINE Messaging API 免費額度約**每月 200～500 則推播**，畢業祝福量在額度內；若爆量可在 LINE Developers Console 查用量。
- 通知只在「家長送出當下」推一次（不發「開始送出」通知，避免洗版）。
- 想暫停通知：把指令碼屬性的 `LINE_CHANNEL_ACCESS_TOKEN` 清空即可，後端會自動跳過、祝福照常寫入。
- 這是**純後端改動**，網站前端（家長看到的頁面）完全不變，不需要重新部署 GitHub Pages。

### ✍️ 卡片一鍵審核（v1.10.1 起）

新祝福卡片下方會多兩個按鈕，直接在 LINE 上審核：

- **✅ 通過公開** → 點了開啟一個審核小頁、把該則設為公開，網站祝福牆數分鐘內同步顯示
- **🙈 維持隱藏** → 維持不公開

審核小頁會顯示該則完整祝福，並附「改為○○」反向按鈕（按錯可即時改回）。整個機制用「按鈕＝開連結」達成，**免 webhook、不影響你其他共用同一個 LINE Bot 的專案**。

> ⚠️ 換新版 `Code.gs` 後**務必重新部署成新版本**（部署 → 管理部署 → 鉛筆 ✏ → 版本「新版本」→ 部署），按鈕才會帶到正確的審核網址。審核用的安全權杖（`MOD_TOKEN`）會在第一次推播時自動產生，**無需額外設定**。
> 之後仍可隨時打開試算表手動改 F 欄審核，兩種方式並存。
